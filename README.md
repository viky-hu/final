# Final

基于 **Monorepo** 架构的 Web 前端项目，通过「组件化拆分」先独立开发各功能窗口，再在主平台中整合。

## 🛠 技术栈

| 类别 | 技术选型 |
|------|----------|
| **管理工具** | pnpm workspaces + Turborepo |
| **主应用 (Host)** | Next.js 14+ (App Router) |
| **开发库 (Lab)** | React + Vite + TypeScript |
| **样式方案** | Tailwind CSS |
| **基础组件** | shadcn/ui (Radix + Tailwind) |
| **动画与特效** | Magic UI + Motion (Framer Motion) |
| **核心动画引擎** | GSAP — 窗口弹出、拖拽、视差滚动 |
| **交互式动画** | Rive — 导航菜单、图标、品牌展示（参考 Dropbox Brand） |
| **组件规范** | Vercel 最佳实践 (Server Components 优先) |

### 为什么选择 GSAP？

GSAP 是 Awwwards 获奖作品的标配，大量顶级网站用它实现丝滑动画。在本项目中：

- **窗口平滑弹出**：通过 GSAP 的 `scale`、`opacity` 和自定义缓动实现
- **拖拽效果**：使用 GSAP Draggable 插件实现可拖拽窗口
- **视差滚动**：使用 GSAP ScrollTrigger 实现视差与滚动联动

### Rive：设计师驱动的交互动画

[Rive](https://rive.app/) 是 Dropbox Brand Guidelines（Awwwards SOTD）等作品使用的交互动画工具。设计师在可视化编辑器中制作动画，导出 `.riv` 文件后在 React 中通过 `@rive-app/react-webgl2` 播放。

- **适用场景**：导航菜单、图标动画、状态机交互、品牌展示模块
- **与 GSAP 分工**：Rive 负责设计感强的交互模块；GSAP 负责窗口级、滚动驱动的动画

## 💡 开发哲学

- **窗口独立性**：每个窗口高内聚，不依赖其他窗口的内部状态
- **配置共享**：UI 风格通过 Tailwind 配置统一管理
- **动画分层**：Rive 负责设计师主导的交互模块；GSAP 负责窗口级、滚动驱动的动画；Magic UI / Motion 负责组件级 UI 动画
- **AI 辅助**：使用 Cursor 进行逻辑生成与 UI 优化

## 第一个窗口：登录界面

### 设计灵感

模仿 **Nav Menu from Dropbox Brand**（Awwwards SOTD），采用「画布思维」：从路径绘制到坍缩，用 **SVG + GSAP** 实现几何动效，文字与线条通过坐标系统精确联动。

---

### 一、完整流程（必读）

```
页面加载 → [自动] 绘制动画 → 阶段 1 结束：介绍界面
                                    ↓
                           用户 scroll 或 click
                                    ↓
              [一次完成] 白蓝反转 + 坍缩 + 切到登录表单
                                    ↓
                    阶段 2 结束：登录表单界面
```

#### 阶段 1：开场绘制（自动播放，无需交互）

| 项目 | 说明 |
|------|------|
| **触发** | 页面加载完成后立即自动播放 |
| **内容** | 四条主干线从边缘划入，Logo 轮廓线同步画出 |
| **结束状态** | 白底、浅蓝网格、四条线围成矩形框，蓝字介绍文案，**蓝色 Logo 在框架左下角**，右下角有「向下」交互提示 |
| **等待** | 动画结束后停留在介绍界面，等待用户 scroll/click |

#### 阶段 2：白蓝反转 + 坍缩（一次 scroll/click 完成）

| 项目 | 说明 |
|------|------|
| **触发** | 用户第一次滚动或点击 |
| **内容** | 一次交互内同时完成：白蓝反转、线条坍缩、内容切换 |
| **白蓝反转** | 框架**内部**：背景变蓝，文字和 Logo 变白 |
| **线条坍缩** | 四条线向中心收缩，**中心点保持不变**（对称收缩） |
| **内容切换** | 介绍文案 → 登录表单 |
| **Logo** | 变白，随框架收缩始终保持在左下角 |
| **结束状态** | 蓝底、白字、白 Logo 左下角，登录表单（邮箱、密码、Sign in 等） |

**核心视觉**：白蓝反转是动画的灵魂，框架内从「白底蓝字」瞬间变为「蓝底白字」，形成强烈对比。

---

### 二、核心动画逻辑：从「绘制」到「坍缩」

#### 第一阶段：路径绘制（Draw Phase）— 自动播放

- **四条主干线**（左竖、右竖、上横、下横）从屏幕边缘划入
- **Logo**（Dropbox 风格，形状与比例正确）轮廓线同步画出，**画完后位于框架左下角**
- 介绍文案随线条裁剪显现
- 动画结束停留在介绍界面

#### 第二阶段：白蓝反转 + 坍缩（Transition Phase）— scroll/click 触发

| 动作 | 说明 |
|------|------|
| 线条坍缩 | 四条线向中心收缩，**中心点固定**，`(x1+x2)/2`、`(y1+y2)/2` 不变 |
| 白蓝反转 | 框架内部背景变蓝，文字和 Logo 变白 |
| 内容切换 | 介绍文案 → 登录表单 |
| 网格显现 | 浅蓝色参考网格在坍缩过程中显现 |
| Logo | 保持左下角，随框架同步缩小 |

---

### 三、画布思维：为什么不用普通 `<div>`？

| 传统 HTML 思维 | 画布思维 |
|----------------|----------|
| `<div>` 是 DOM 中的矩形块 | 文字是画布上的「动态层」 |
| 难以随线条的缩放、移动、切割同步形变 | 边界完全由运动的线条实时裁剪 |
| 用「盒子」套「动态的线」会失败 | 文字必须直接「长」在 SVG 内部 |

**实现方式**：使用 **SVG `<foreignObject>`** 或 **SVG `<text>`**，保证文字与线条绝对同步。

---

### 四、技术实现：SVG + GSAP

#### 坐标关联系统

定义四个动态变量：`x1`, `x2`, `y1`, `y2`

| 变量 | 含义 |
|------|------|
| x1 | 左竖线位置 |
| x2 | 右竖线位置 |
| y1 | 上横线位置 |
| y2 | 下横线位置 |

**中心点不变**：坍缩时 `(x1+x2)/2`、`(y1+y2)/2` 保持不变，左右、上下对称收缩。

**Logo**：位置基于 `x1`, `y2` 计算，始终锚定框架左下角；形状与比例参考 Dropbox Logo。

#### GSAP Timeline 编排

1. **阶段 1（自动）**：`fromTo` + `stroke-dashoffset` 绘制四条主干线 + Logo 轮廓线，绘制完成即停留在介绍界面
2. **阶段 2（scroll/click 触发）**：
   - `to` 动画 `x1`, `x2`, `y1`, `y2`，**中心点不变**，实现对称坍缩
   - 框架内部背景色白→蓝，文字/Logo 蓝→白
   - 内容从介绍文案切换为登录表单
3. **文字遮罩**：`<clipPath>` 矩形绑定 `x1`, `x2`, `y1`, `y2`，文字显现范围精准跟随线条

#### 触发方式

- **阶段 1**：页面加载后自动播放，无需交互
- **阶段 2**：一次滚轮或一次点击/触摸 → 完整播放白蓝反转 + 坍缩 + 登录表单切入
- 不使用 ScrollTrigger scrub

---

### 五、登录表单：阶段 2 结束后的内容

坍缩到位后，框架内显示登录表单（邮箱、密码、记住我、忘记密码、Sign in、注册入口）。表单采用悬浮标签 + 底部线框的高级呈现方式。Logo 保持白色，位于框架左下角。

---

### 六、实现清单（供 Cursor 按序执行）

1. 创建 SVG 画布，定义 `x1`, `x2`, `y1`, `y2` 为可动画变量
2. 四条主干线（`<line>` 或 `<path>`）绑定上述坐标
3. Logo 用 `<path>` 绘制 Dropbox 风格轮廓，**形状与比例正确**，**画完后位于框架左下角**（基于 `x1`, `y2`）
4. **阶段 1**：`useLayoutEffect` 中自动 `tl.play()`，GSAP `fromTo` + `stroke-dashoffset` 实现路径绘制，结束后停留在介绍界面
5. **阶段 2**：监听 `wheel` / `click`，GSAP `to` 动画 `x1,x2,y1,y2`，**中心点不变**，实现对称坍缩
6. 白蓝反转：框架内背景变蓝，文字和 Logo 变白
7. 内容切换：介绍文案 → 登录表单
8. 浅蓝色参考网格在坍缩过程中显现
9. 文字用 `<foreignObject>` 或 `<text>` + `<clipPath>`，clipPath 矩形绑定 `x1,x2,y1,y2`
