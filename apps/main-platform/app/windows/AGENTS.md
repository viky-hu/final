# AGENTS.md（app/windows）

## 作用域
本文件作用于 `apps/main-platform/app/windows` 目录树。

## 总体原则
1. 每个窗口目录是一个独立模块边界。
2. 跨窗口能力优先通过 `shared` 组件与明确的 props 契约共享。
3. 禁止在一个窗口内直接修改另一个窗口的内部状态实现。

## 文件职责约束
- `*/<WindowName>Window.tsx`：窗口级编排、状态汇聚、子组件装配。
- `*/components/*.tsx`：局部视图与交互实现。
- `shared/*.tsx`：可复用 UI/交互能力，不能耦合单窗口业务私有状态。

## 新功能落地要求
- 必须先写清“主责文件 + 协同文件”。
- 若新增状态，必须声明状态唯一来源（single source of truth）。
- 若影响多个窗口，必须评估是否抽象到 `shared`。

## 扩展性审查
每次功能合入前至少回答：
- 再增加一个窗口是否可复用？
- 再增加一个后端模式是否需要大改？
- 是否存在明显重复代码可当次收敛？
