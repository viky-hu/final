# AGENTS.md（app/api）

## 作用域
本文件作用于 `apps/main-platform/app/api` 目录树。

## 目标
将前端与外部服务隔离在 BFF 层，统一输入校验、错误归一化、鉴权与限流策略。

## Route Handler 规则
1. 路由文件仅做：解析请求、校验、调用服务、返回标准响应。
2. 复杂业务逻辑放到服务层（`lib/server/**` 或同级 server 工具文件）。
3. 每个接口必须提供稳定的错误语义（HTTP 状态码 + 可读错误消息）。
4. 所有密钥与后端地址来自环境变量，禁止写死。
5. 任何敏感变量不得使用 `NEXT_PUBLIC_*`。

## 扩展性规则
- 新增接口时，必须说明未来如何支持：
  - 新节点接入；
  - 新字段向后兼容；
  - 超时/失败降级。
- 如出现重复响应结构，必须抽象公共响应构造器。

## 安全与治理
- 默认要求：鉴权、输入校验、授权、限流、错误处理。
- 发现仅“能跑通但无边界保护”的实现，应视为未完成。
